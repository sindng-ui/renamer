# ⚡ 마지막 작업 원클릭 재실행 (Quick Run View) 구현 계획서 🗺️

형님! 지난번 성공했던 리네임 작업을 앱이 기억해두었다가, 다음번 실행 시 화면 중앙에 아주 누르고 싶게 생긴 매력적인 대형 네온 버튼 하나만 제공하여 원클릭으로 모든 변환 작업을 마칠 수 있도록 하는 핵심 UX 기능을 설계했습니다.

이 기능은 3,000개 이상의 대용량 파일 변경 시에도 매끄럽게 돌아가며, 60fps 보장을 위해 성능을 저하시키는 blur 계열 필터는 일체 배제하고 순수 box-shadow와 그라데이션만으로 구성된 프리미엄 다크-네온 스타일로 꾸밀 예정입니다.

---

## 🙋 유저 검토 사항 (User Review Required)
> [!IMPORTANT]
> - **작업 기억 기준**: 변환 작업 중 **최소 1개 이상의 파일이 성공적으로 변경 완료된 시점**에 해당 작업의 정보(폴더 경로, 파일 개수, 변환 규칙 설정값)를 `localStorage`에 기록합니다.
> - **작업 자동 복구**: 다음 기동 시, 해당 정보가 있으면 다른 복잡한 UI 요소를 숨기고 오직 매력적인 대형 버튼을 중앙에 띄웁니다.
> - **원클릭 흐름**: 버튼을 탭하면 앱이 백그라운드에서 즉시 해당 폴더의 최신 파일들을 다시 읽어오며(List Files), 읽기가 완료되자마자 당시 규칙을 자동으로 적용해 리네임을 즉시 실행합니다. (미리보기 화면을 거치지 않아 극도로 빠릅니다!)
> - **상세 페이지 이동**: 아래쪽에 제공될 "상세 설정 및 다른 작업" 버튼을 누르면 언제든지 기존 상세 설정 및 미리보기 화면으로 복귀합니다.

## 🧪 검증 계획 (Verification Plan)

### 수동 검증 (Manual Verification)
1. **앱 최초 진입 및 세부 규칙 변경**: 파일들을 임의의 폴더에서 지정하고 이름을 무작위(Random) 또는 접미사/접두사로 성공적으로 변환합니다.
2. **작업 기억 검증**: 변환이 끝난 후 앱을 새로고침(혹은 재시작)하여 화면에 거대한 네온 버튼이 활성화된 퀵 실행 뷰가 제일 먼저 나타나는지 확인합니다.
3. **퀵 버튼 정보 검증**: 버튼 내부 및 카드 영역에 "📂 {마지막 폴더명}", "📄 {개수}개 파일", "🎲 {변환 옵션 요약}" 정보가 예쁘고 간결하게 표시되는지 확인합니다.
4. **원클릭 재실행**: 거대 네온 버튼을 클릭했을 때, 즉각 로딩 상태로 진입하고 해당 폴더에서 파일을 긁어와 리네임 처리를 백그라운드로 쾌적하게 마치는가 검증합니다.
5. **분기 경로 검증**: "다른 작업 하기" 버튼을 누르면 즉시 상세 페이지로 넘어가며, 즐겨찾기 및 기능이 정상 작동하는지 확인합니다.
